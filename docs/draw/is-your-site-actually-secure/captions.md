## Slide 2 — SSL / HTTPS

**Caption:**
Check 1: SSL / HTTPS.

Without that padlock, everything your users send — passwords, card details, personal data — is visible to anyone on the network. 82% of users will leave a site that shows "Not Secure".

Fix: Install SSL (free via Let's Encrypt), force HTTPS with a 301, set HSTS headers.

## Slide 3 — Mixed Content

**Caption:**
Check 2: Mixed content.

Your page loads over HTTPS but some resources still load over HTTP. The padlock disappears and browsers may block those resources entirely.

Fix: Update all URLs to HTTPS. Add a Content-Security-Policy header with upgrade-insecure-requests.

## Slide 4 — Exposed Admin Panels

**Caption:**
Check 3: Exposed admin panels.

/wp-admin, /login, /administrator — automated bots scan for these constantly. If yours is publicly accessible with no rate limiting, it's only a matter of time.

90,000+ WordPress sites are attacked per minute.

Fix: Change default URLs, add IP restrictions, enable 2FA.

## Slide 5 — Outdated CMS

**Caption:**
Check 4: Outdated CMS.

Running an old version of WordPress, Drupal, or Joomla? Every known vulnerability is publicly documented. Attackers just need to check your version number.

56% of CMS-based sites run outdated software.

Fix: Enable auto-updates. Remove unused plugins. Upgrade PHP to 8.1+.

## Slide 6 — Sensitive File Exposure

**Caption:**
Check 5: Sensitive file exposure.

.env files, wp-config.php, .git directories, database backups — if any of these are publicly accessible, attackers have your database passwords and API keys in plain text.

Fix: Block access in your web server config. Add to .gitignore. Run a scan to check.
