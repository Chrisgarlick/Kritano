Sensitive File Exposure

Configuration files, backups, and environment variables left publicly accessible. One wrong file and attackers have your database credentials.

Files That Should Never Be Public

.env wp-config.php .git/ database.yml *.sql backups .DS_Store
Fix

Block access to sensitive files in your web server config. Add them to .gitignore. Use environment variables instead of config files in web root. Run a scan to check what's publicly accessible.

Real risk: Exposed .env files contain database passwords, API keys, and secret tokens in plain text.
