# PDF Branding Tiers — Implementation Plan

## Context

PDFs currently use site branding if available but with no tier gating — any user can export and any verified site gets custom branding. The `export_pdf` and `white_label` flags exist in `tier_limits` but are never enforced.

**Goal**: Introduce tiered PDF branding:
- **Free**: No PDF export (403)
- **Starter/Pro**: Per-site branding (company name, colors) but footer always says "Generated by PagePulser", no logo
- **Agency/Enterprise**: Full white-label — org-level branding as default across all sites, per-site overrides, custom footer text (e.g. "Prepared by Agency X"), logo rendered in PDF header
- **Tier inheritance**: Org tier applies for org sites only; personal sites use personal subscription tier

---

## Implementation Steps

### 1. Update Organization Types
**File**: `server/src/types/organization.types.ts`

Add `OrgBranding` interface and extend `OrganizationSettings`:

```typescript
export interface OrgBranding {
  companyName?: string;
  logoUrl?: string;
  primaryColor?: string;
  secondaryColor?: string;
  accentColor?: string;
  footerText?: string; // "Prepared by Agency X"
}

// Add to OrganizationSettings:
branding?: OrgBranding;
```

### 2. Update Site Types
**File**: `server/src/types/site.types.ts` (line 44-51)

Add `footerText` to existing site branding interface:

```typescript
branding?: {
  companyName?: string;

  secondaryColor?: string;
  accentColor?: string;
```

### 3. Create PDF Branding Resolution Service (NEW)
**File**: `server/src/services/pdf-branding.service.ts`

Exports `resolvePdfBranding(pool, siteId, userId) → ResolvedBranding`

Resolution chain:
1. Get tier via `getSiteOwnerTierLimits(siteId)` or `getUserTierLimits(userId)`
2. `export_pdf === false` (free) → return defaults with `canExportPdf: false`
3. `white_label === false` (starter/pro) → site colors + name, footer = "Generated by PagePulser", no logo
4. `white_label === true` (agency+) → load org branding (via site's org), merge site overrides on top, custom footer, logo

```typescript
export interface ResolvedBranding {
  companyName: string;       // default: 'PagePulser'
  logoUrl: string | null;    // only non-null for agency+
  primaryColor: string;      // default: '#4f46e5'
  secondaryColor: string;    // default: '#6366f1'
  accentColor: string;       // default: '#f59e0b'
  footerText: string;        // default: 'Generated by PagePulser'
  canExportPdf: boolean;     // false for free
  canWhiteLabel: boolean;    // true for agency+
}
```

Reuses existing functions:
- `getSiteOwnerTierLimits(siteId)` from `server/src/services/site.service.ts`
- `getUserTierLimits(userId)` from `server/src/services/site.service.ts`

### 4. Refactor PDF Export Endpoint
**File**: `server/src/routes/audits/index.ts` (lines 1474-2070)

**a. Tier gate** (after line 1493):
- Call `resolvePdfBranding(pool, audit.site_id, userId)`
- If `!canExportPdf` → 403 "PDF export requires Starter plan or higher"

**b. Remove inline branding lookup** (lines 1496-1521):
- Delete the `SiteBranding` interface and manual query block — replaced by the service

**c. Replace branding variables** (lines 1563-1605):
- `brandName = branding.companyName`
- `brandLogo = branding.canWhiteLabel ? branding.logoUrl : null`
- `colors.primary/secondary/accent` from `branding.*`

**d. Render logo in cover page header** (after line 1691):
- If `brandLogo` is set, fetch the image via `fetch()` with 5s timeout
- Render it in the header area (left side, 30px height)
- Graceful fallback if fetch fails

**e. Update footer** (line 2056):
- Replace `Generated by ${brandName}` with `branding.footerText`

### 5. Update Organization Settings Validation
**File**: `server/src/routes/organizations/index.ts`

- Extend `updateOrgSchema` Zod to accept `settings.branding` with: companyName, logoUrl, primaryColor, secondaryColor, accentColor, footerText
- In the PATCH handler: if `settings.branding` is present, check org tier — if `white_label === false`, reject with 403

### 6. Update docs/TIERS.md
Add a "PDF Branding" row to the capabilities table.

## Verification

1. **Free user** exports PDF → 403 with clear message
2. **Starter user** with site branding → PDF uses site colors + name, footer says "Generated by PagePulser", no logo
3. **Agency user** with org branding, no site branding → PDF uses org colors/name/logo/footer
4. **Agency user** with org branding AND site branding → site fields override org where set
5. **Agency user** with no branding → PagePulser defaults but custom footer available
6. **Unverified site** on starter+ → no branding applied (existing behavior)
7. **Logo URL that 404s** → PDF generates without logo (graceful fallback)
8. **Org branding update** on free/starter/pro tier → 403 error
